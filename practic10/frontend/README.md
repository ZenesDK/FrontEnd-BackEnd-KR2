# Практическая работа №10: Хранение токенов на фронтенде

## Цель работы
Реализовать клиентское React-приложение, которое взаимодействует с защищённым API (JWT + refresh-токены). Научиться хранить токены на клиенте (localStorage), автоматически добавлять их в запросы и обрабатывать истечение access-токена с помощью axios-interceptors.

## Функциональность
- Регистрация и вход пользователя
- После входа токены (access и refresh) сохраняются в localStorage
- Все запросы к защищённым ресурсам (товары) автоматически получают заголовок Authorization: Bearer <accessToken>
- При ответе 401 (Unauthorized) автоматически отправляется запрос на обновление токенов (POST /auth/refresh)
- Если обновление прошло успешно, исходный запрос повторяется с новым токеном
- Если обновление не удалось (refresh токен истёк или невалиден), пользователь перенаправляется на страницу входа
- Полноценный CRUD для товаров (просмотр списка, создание, редактирование, удаление)

## Технологии
- React (Create React App)
- React Router v6
- Axios (с перехватчиками)
- Сервер: Node.js + Express (из практики №9)

## Запуск проекта

### Предварительные требования
- Node.js (версия 14+)
- npm или yarn

### 1. Запуск сервера
Серверная часть находится в папке `backend`. Скопируйте туда код из практической работы №9 (или используйте уже готовый сервер). Убедитесь, что сервер разрешает CORS-запросы с порта 3001.

```bash
cd practice-10/backend
npm install
npm run dev
```

Сервер будет доступен на http://localhost:3000

### 2. Запуск клиента
```bash
cd practice-10/frontend
npm install
npm start
```

Клиент откроется на http://localhost:3001

## Использование

1. Перейдите на /register и создайте нового пользователя.
2. Войдите на /login – после успешного входа вы будете перенаправлены на список товаров.
3. Добавляйте, редактируйте и удаляйте товары. Все операции требуют наличия валидного access-токена.
4. Если access-токен истечёт, axios автоматически обновит его и повторит запрос. Вы не заметите этого процесса.

## Примечания по безопасности

- В демонстрационных целях токены хранятся в localStorage. В реальных проектах для большей безопасности рекомендуется хранить refresh-токен в httpOnly cookie, а access-токен – в памяти приложения.
- Перехватчик ответов настроен так, чтобы при ошибке обновления токена очищать хранилище и перенаправлять на логин.

## Список маршрутов API, используемых клиентом
```
Метод	Маршрут	                Описание
POST	/api/auth/register	    Регистрация
POST	/api/auth/login	        Вход, получение токенов
POST	/api/auth/refresh	    Обновление пары токенов
GET     /api/auth/me	        Информация о текущем пользователе
GET	    /api/products	        Список товаров
POST	/api/products	        Создать товар
GET	    /api/products/:id	    Получить товар по ID
PUT	    /api/products/:id	    Обновить товар
DELETE	/api/products/:id	    Удалить товар
```
